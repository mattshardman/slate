    Introduction

    Welcome to the Gigapet API.

    This is example API documentation. The github repo can be found here: github repo

    And the api endpoints can be found here: api

    Authentication

    Gigapet uses Json Web Tokens (jwt) to authenticate users. To access endpoints send your jwt as the Authorization header.

    Authorization: [your jwt here]

    You must replace [your jwt here] with your personal jwt.

    Register a user

    To register a user, use this code:

    axios.post(
  "https://lambda-gigapet.herokuapp.com/api/auth/register", {
    name: String, // required
    email: String, // required
    username: String, // required
    password: String, // required
    pin: String, // not required 
    img_url: String, // not required 
});

    This will return the following JSON response:

      // response HTTP 201
  {
    id: Number,
    jwt: String
  }

    It's recommended to store the jwt in local storage

    HTTP Request

    To register a new user send a POST request to:

    POST https://lambda-gigapet.herokuapp.com/api/auth/register

    Login a user

    To register a user, use this code:

    axios.post(
  "https://lambda-gigapet.herokuapp.com/api/auth/login", {
    username: String, // required
    password: String, // required
});

    This will return the following JSON response:

      // response HTTP 200
  {
    id: Number,
    jwt: String
  }

    It's recommended to store the jwt in local storage

    HTTP Request

    To login an existing user send a POST request to:

    POST https://lambda-gigapet.herokuapp.com/api/auth/login

    Query food entry

    To get all CARBS food entries for the WEEK for child with id 1, use this code:

    axios.defaults.headers.common["Authorization"] = "your jwt";

axios.get("https://lambda-gigapet.herokuapp.com/api/child/1/entries?category=carbs&time_span=week");

    The above command returns JSON structured like this:

    {
  "carbs": [{
            "id": 1,
            "name": "Porridge",
            "quantity": 2,
            "meal": "Breakfast",
            "category": "Carbs",
            "date_added": "2019--04-15",
            "date_update": "2019--04-15",
            "child_id": 1
          }],
}

    This endpoint returns all food entries for a specific child, within given query parameters.

    Query Parameters

    Parameter Default Options Description
    category null protein, carbs, diary, vegetables, fruit, treats If not set, will return all categories
    time_span null day, week, month If not set, will return all entries
    specific_time_span null 2019-04-09T11:45:12.000Zto2019-04-17T11:45:12.000Z date can be in ISO format or UNIX timestamp

    HTTP Request

    GET "https://lambda-gigapet.herokuapp.com/api/child/1/entries?category={choose a category}&time-span={choose a time span}&specific_time_span={(date)to(date)}"

    Remember — a valid jwt must be sent as an Authorization header!
    Warning - time_span & specific_time_span should not be used together!

    Errors

    The Gigapet API uses the following error codes:

    Error Code Meaning
    400 Bad Request -- Your request is invalid.
    401 Unauthorized -- Your JWT is invalid.
    403 Forbidden -- The kitten requested is hidden for administrators only.
    404 Not Found -- The specified route could not be found.
    405 Method Not Allowed -- You tried to access a route with an invalid method.
    406 Not Acceptable -- You requested a format that isn't json.
    410 Gone -- The requested has been removed from our servers.
    429 Too Many Requests -- You're requesting too many things! Slow down!
    500 Internal Server Error -- We had a problem with our server. Try again later.
    503 Service Unavailable -- We're temporarily offline for maintenance. Please try again later.
